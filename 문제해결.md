# 문제 해결 가이드

## 앱이 실행되지 않는 경우

### 1. 의존성 패키지가 설치되지 않은 경우

**해결 방법:**

1. **방법 A: 배치 파일 사용 (가장 쉬움)**
   - `SETUP.bat` 파일을 더블클릭하여 실행

2. **방법 B: PowerShell에서 직접 실행**
   ```
   npm install
   npm run dev
   ```

3. **방법 C: PowerShell 실행 정책 문제인 경우**
   - PowerShell을 **관리자 권한**으로 실행
   - 다음 명령어 입력:
   ```powershell
   Set-ExecutionPolicy RemoteSigned
   ```
   - 그 다음:
   ```powershell
   npm install
   npm run dev
   ```

### 2. 포트 3000이 이미 사용 중인 경우

**해결 방법:**

다른 포트로 실행:
```powershell
$env:PORT=3001; npm run dev
```

또는 `package.json`의 dev 스크립트를 수정:
```json
"dev": "next dev -p 3001"
```

### 3. 브라우저에서 "localhost에 연결할 수 없음" 오류

**확인 사항:**
- 개발 서버가 실행 중인지 확인
- 터미널에 "Ready on http://localhost:3000" 메시지가 보이는지 확인
- 올바른 URL 사용: `http://localhost:3000` (https가 아님)

### 4. 빌드 오류가 발생하는 경우

**해결 방법:**

1. `node_modules` 폴더 삭제
2. `package-lock.json` 삭제 (있다면)
3. 다시 설치:
   ```powershell
   npm install
   ```
4. 빌드 테스트:
   ```powershell
   npm run build
   ```

### 5. TypeScript 오류

**해결 방법:**

1. TypeScript 재설치:
   ```powershell
   npm install --save-dev typescript @types/node @types/react @types/react-dom
   ```

### 6. Tailwind CSS 스타일이 적용되지 않는 경우

**해결 방법:**

1. `postcss.config.js` 파일 확인
2. `tailwind.config.js` 파일 확인
3. `app/globals.css`에 `@tailwind` 지시문이 있는지 확인

### 7. 로컬 스토리지 오류 (브라우저 콘솔)

**해결 방법:**
- 브라우저의 로컬 스토리지가 활성화되어 있는지 확인
- 시크릿 모드가 아닌 일반 브라우저 모드 사용

## 빠른 진단

다음 명령어로 문제를 확인하세요:

```powershell
# Node.js 버전 확인
node --version

# npm 버전 확인
npm --version

# 의존성 설치 상태 확인
Test-Path node_modules

# 빌드 테스트
npm run build
```

## 여전히 문제가 있나요?

1. 터미널의 전체 오류 메시지를 확인하세요
2. 브라우저 개발자 도구(F12)의 콘솔 오류 확인
3. `package.json`의 스크립트가 올바른지 확인

## 연락처

문제를 해결할 수 없다면 다음 정보와 함께 문의하세요:
- 운영체제 버전
- Node.js 버전
- 오류 메시지 전체
- 터미널 출력 전체
